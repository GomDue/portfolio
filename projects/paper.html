기획 배경

  • 피처 엔지니어링: RFECV로 중요 피처 선별, 재무비율(부채/유동/영업이익률 등) 생성
  • 불균형 처리: SMOTE로 분식/정상 클래스 균형화
  • 모델링: ANN/RF 학습 및 비교, MinMaxScaler 정규화
  • 튜닝: GridSearchCV·RandomizedSearchCV로 RF 하이퍼파라미터 최적화
  • 평가: 5-Fold 교차검증으로 Accuracy·Recall·F1 종합 평가

  1. 개발 상세
3.1. 데이터 수집·정제 배경

배경

  • 기업 분식회계 탐지를 위해서는 정확하고 신뢰성 있는 재무 데이터와 분식 여부 라벨이 필수였습니다.
  • 데이터를 직접 생성할 수 없기 때문에, 공신력 있는 기관 데이터를 활용해 재현성과 신뢰성을 확보해야 했습니다.

도전

  • 재무제표 항목명은 기업마다 다르고, 공시 연도에 따라 표기 방식이 조금씩 달라서 데이터 불일치 문제가 있었습니다.
  • 여러 해에 걸쳐 동일 기업 데이터를 합치면서 누락 항목 처리와 중복 제거도 필요했습니다.

해결 방안

  • 데이터 출처:
    • DART(전자공시시스템): 재무상태표, 손익계산서, 현금흐름표
    • FSS(금융감독원): 분식 여부 레이블
  • 정제 방법:
    • 항목명 표준화, 중복 레코드 제거
    • 연도별 데이터 병합 및 NaN 처리
    • 분석과 무관한 텍스트/메타데이터 컬럼 제거

성과

  • 분석에 필요한 정량 재무 데이터 + 신뢰성 있는 라벨을 확보했습니다.
  • 가공 전 435개 변수 → 표준화된 구조로 변환을 완료했습니다.
3.2. 전처리 설계 이유

배경

  • 원본 데이터(435개 변수)는 너무 많아, 불필요한 피처를 제거하지 않으면 학습 속도가 느려지고 과적합 가능성이 높았습니다.
  • 또한 ANN(인공신경망)과 RF(랜덤포레스트)는 입력 데이터의 스케일과 특성에 따라 성능 차이가 발생하기 때문에 피처 가공이 필요했습니다.

도전

  • 변수 개수가 많아 모델 복잡도가 증가하고 해석이 어려웠습니다.
  • 분식 기업 비율이 매우 낮아 학습 시 불균형 문제가 발생하였고, 이로 인해 Recall(재현율)이 저하될 우려가 있었습니다.

해결 방안

  • 피처 선택: RFECV(Recursive Feature Elimination with Cross-Validation)을 적용하여 상위 약 100개의 중요한 피처만 선별하였습니다.
  • 파생 지표 생성: 절대값 대신 부채비율, 유동비율, 영업이익률 등 의미 있는 재무비율을 생성하였습니다.
  • 정규화: ANN과 RF가 모두 안정적으로 학습될 수 있도록 MinMaxScaler로 입력 스케일을 조정하였습니다.
  • 불균형 처리: SMOTE(Synthetic Minority Oversampling Technique)를 적용하여 소수 클래스(분식 기업)를 오버샘플링하였습니다.

성과

  • 불필요한 변수를 제거하고 중요한 재무 지표 중심으로 데이터를 구성하여 모델 학습 효율을 향상시켰습니다.
  • SMOTE를 통해 클래스 불균형을 해소하여 Recall 저하를 방지하였고, 모델이 분식 기업을 더 잘 탐지할 수 있도록 개선하였습니다.
3.3. 모델 최적화 필요성

배경

  • 초기 모델 학습에서는 정확도는 높았지만 Recall이 낮아, 실제 분식 기업을 놓치는 비율이 높았습니다.
  • 금융 사기 탐지에서는 “적발하지 못하는 것”이 더 큰 문제이므로, Recall 개선이 반드시 필요했습니다.

도전

  • ANN과 RF 모두 기본 하이퍼파라미터로는 Recall을 충분히 확보하지 못하였습니다.
  • 과적합을 방지하면서 동시에 성능을 향상시켜야 하는 과제가 있었습니다.

해결 방안

  • Random Forest 튜닝
    • GridSearchCV와 RandomizedSearchCV를 활용하여 트리 수(n_estimators), 최대 깊이(max_depth), 분할 기준(max_features)을 조정하였습니다.
  • ANN 튜닝
    • 은닉층 수, 뉴런 수, 학습률, 활성화 함수를 변경하며 최적 구조를 탐색하였습니다.
    • Dropout 기법을 적용하여 과적합을 방지하였습니다.
  • 평가지표 관리
    • Accuracy뿐만 아니라 Recall과 F1-score를 함께 모니터링하여 모델 개선 방향을 설정하였습니다.

성과

  • Recall을 ANN 87%, RF 85%까지 끌어올릴 수 있었습니다.
  • ANN은 분식 탐지 성능(Recall)에서 우수하였고, RF는 해석 용이성과 빠른 학습 속도가 장점으로 확인되었습니다.
  • 피처 중요도 분석 결과, 부채비율·유동비율·영업이익률이 핵심 지표로 도출되었습니다.
3.4. 실험 결과 및 분석

실험 결과 (성능 지표)

모델 F1-score (정상) F1-score (분식) Accuracy
NN 0.73 0.73 0.73
RF (GridSearch) 0.71 0.71 0.71
RF (RandomSearch) 0.70 0.72 0.71
RF (RFECV) 0.72 0.81 0.77
RF (SMOTE) 0.77 0.77 0.77

성능 비교

  • NN(인공신경망): 정상과 분식 모두 F1-score가 0.73으로 균형 잡힌 성능을 보였으며, Accuracy는 0.73이었습니다.
  • RF (GridSearch): NN보다 F1-score와 Accuracy가 다소 낮아 0.71 수준이었습니다.
  • RF (RandomSearch): GridSearch와 유사하였으나, 분식 클래스의 F1-score가 0.72로 약간 개선되었습니다.
  • RF (RFECV): 변수 선택을 적용하였을 때 분식 클래스의 F1-score가 0.81로 크게 향상되었고, Accuracy는 0.77로 가장 높았습니다.
  • RF (SMOTE): 정상과 분식 모두 F1-score가 0.77로 균형을 이루었으며, Accuracy도 0.77로 RFECV와 동일한 성능을 보였습니다.

분석 결과

  • Recall과 Precision의 균형이 중요한 분식 탐지에서 RFECV와 SMOTE가 뚜렷한 효과를 보였습니다.
  • RFECV는 불필요한 변수를 제거하고 핵심 피처만 학습하게 하여 분식 탐지 성능을 크게 향상시켰습니다.
  • SMOTE는 데이터 불균형을 완화하여 두 클래스 간 성능 균형을 확보하였습니다.
  • NN은 분식과 정상 성능이 비슷하였지만, 절대 수치에서는 RF(RFECV/SMOTE)보다 낮은 결과를 보였습니다.
  • RF는 해석 용이성과 학습 속도에서 강점이 있었지만, 전처리 없이 기본 튜닝만으로는 분식 탐지 성능이 부족하였습니다.

결론

  • 분식회계 탐지에서는 단순한 하이퍼파라미터 튜닝보다 변수 선택(RFECV)과 불균형 처리(SMOTE)가 성능 개선에 결정적인 역할을 하였습니다.
  • NN은 기본적으로 균형 잡힌 성능을 보였으나, RF에 적절한 전처리를 적용할 경우 더 높은 성능을 달성할 수 있었습니다.
  • 최종적으로:
    • 분식 탐지율 극대화가 필요할 경우 → RFECV + RF
    • 클래스 간 균형 유지가 필요할 경우 → SMOTE + RF
