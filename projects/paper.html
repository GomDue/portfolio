  1. 프로젝트 개요
기획 배경
  • ESG 경영의 중요성으로 기업 분식회계 탐지에 대한 요구가 커지며, 신뢰 가능한 재무 데이터 기반의 판별 모델 개발 필요가 대두되었습니다.
프로젝트 목표
  • 기업 재무제표 데이터를 활용해 분식 여부 탐지 모델을 개발
  • 인공신경망(ANN)과 랜덤포레스트(RF) 성능 비교 및 최적화
  • 데이터 불균형(SMOTE)과 피처 선택(RFECV)으로 탐지율(Recall) 향상
팀 구성
  • 2인 (지도 교수 1, 학부 연구생 1)
주요 기능
  • 데이터 수집·정제: DART/FSS에서 재무제표·라벨 수집, 표준화/병합/결측 처리
  • 피처 엔지니어링: RFECV로 중요 피처 선별, 재무비율(부채/유동/영업이익률 등) 생성
  • 불균형 처리: SMOTE로 분식/정상 클래스 균형화
  • 모델링: ANN/RF 학습 및 비교, MinMaxScaler 정규화
  • 튜닝: GridSearchCV·RandomizedSearchCV로 RF 하이퍼파라미터 최적화
  • 평가: 5-Fold 교차검증으로 Accuracy·Recall·F1 종합 평가

  1. 데이터 수집·정제 배경
배경
  • 기업 분식회계 탐지를 위해서는 정확하고 신뢰성 있는 재무 데이터와 분식 여부 라벨이 필수였습니다.
  • 데이터를 직접 생성할 수 없기 때문에, 공신력 있는 기관 데이터를 활용해 재현성과 신뢰성을 확보해야 했습니다.
도전
  • 재무제표 항목명은 기업마다 다르고, 공시 연도에 따라 표기 방식이 조금씩 달라서 데이터 불일치 문제가 있었습니다.
  • 여러 해에 걸쳐 동일 기업 데이터를 합치면서 누락 항목 처리와 중복 제거도 필요했습니다.
해결 방안
  • 데이터 출처:
    • DART(전자공시시스템): 재무상태표, 손익계산서, 현금흐름표
    • FSS(금융감독원): 분식 여부 레이블
  • 정제 방법:
    • 항목명 표준화, 중복 레코드 제거
    • 연도별 데이터 병합 및 NaN 처리
    • 분석과 무관한 텍스트/메타데이터 컬럼 제거
성과
  • 분석에 필요한 정량 재무 데이터 + 신뢰성 있는 라벨을 확보했고,
  • 가공 전 435개 변수 → 표준화된 구조로 변환 완료.
  1. 전처리 설계 이유
배경
  • 원본 데이터(435개 변수)는 너무 많아서, 불필요한 피처를 제거하지 않으면 학습 속도가 느리고 과적합 가능성이 높았습니다.
  • 또한 ANN(인공신경망)과 RF(랜덤포레스트)는 입력 데이터 스케일과 특성에 따라 성능 차이가 나기 때문에 피처 가공이 필요했습니다.
도전
  • 변수 개수가 많아 모델 복잡도 증가 및 해석 어려움
  • 분식 기업 비율이 매우 낮아 학습 시 불균형 문제 발생 → Recall(재현율) 저하
해결 방안
  • 피처 선택: RFECV(Recursive Feature Elimination with Cross-Validation)로 상위 100여 개 중요한 피처만 선별
  • 파생 지표 생성: 절대값 대신 부채비율, 유동비율, 영업이익률 등 의미 있는 재무비율 생성
  • 정규화: ANN·RF 모두 잘 학습되도록 MinMaxScaler로 스케일 조정
  • 불균형 처리: SMOTE(Synthetic Minority Oversampling Technique)로 소수 클래스(분식 기업) 오버샘플링
성과
  • 불필요한 변수를 제거하고 중요한 재무 지표 중심으로 구성 → 모델 학습 효율 향상
  • SMOTE로 클래스 균형을 맞춰 Recall 저하 방지
  1. 모델 최적화 필요성
배경
  • 초기 모델 학습에서는 정확도는 높았지만, Recall이 낮아 실제 분식 기업을 놓치는 비율이 높았습니다.
  • 금융 사기 탐지에서는 “적발하지 못하는 것”이 더 큰 문제이므로 Recall 개선이 필요했습니다.
도전
  • ANN과 RF 모두 기본 하이퍼파라미터로는 Recall이 충분히 확보되지 않음
  • 과적합 방지와 성능 향상을 동시에 달성해야 함
해결 방안
  • Random Forest 튜닝:
    • GridSearchCV / RandomizedSearchCV로 트리 수(n_estimators), 최대 깊이(max_depth), 분할 기준(max_features) 조정
  • ANN 튜닝:
    • 은닉층 수, 뉴런 수, 학습률, 활성화 함수 변경
    • Dropout으로 과적합 방지
  • 평가지표: Accuracy뿐 아니라 Recall과 F1-score를 함께 모니터링
성과
  • Recall을 ANN 87%, RF 85%까지 끌어올림
  • ANN은 분식 탐지 성능(Recall)에서 우수, RF는 해석 용이성과 빠른 학습이 장점
  • 피처 중요도 분석 결과 부채비율, 유동비율, 영업이익률이 핵심 지표로 도출

성능 비교
  • NN(인공신경망): 정상/분식 모두 F1-score가 0.73로 균형 잡힌 성능, Accuracy 0.73
  • RF (GridSearch): NN보다 F1-score와 Accuracy가 다소 낮음(0.71)
  • RF (RandomSearch): GridSearch와 유사하나, 분식 클래스의 F1-score가 0.72로 약간 개선
  • RF (RFECV): 변수 선택을 적용하니 분식 클래스 F1-score가 0.81로 크게 향상, Accuracy 0.77로 가장 높음
  • RF (SMOTE): 정상/분식 모두 F1-score가 0.77로 균형, Accuracy 0.77로 RFECV와 동일
분석 결과
  • Recall 및 Precision 균형이 중요한 분식 탐지에서 RFECV와 SMOTE가 뚜렷한 효과를 보임
  • RFECV → 불필요한 변수를 제거하고 핵심 피처만 학습하여 분식 탐지 성능 향상
  • SMOTE → 데이터 불균형을 완화해 두 클래스 간 성능 균형 확보
  • NN은 분식/정상 성능이 비슷하지만, 절대 수치에서는 RF(RFECV/SMOTE)보다 낮음
  • RF는 해석 용이성과 속도에서 강점이 있지만, 전처리 없이 기본 튜닝만 하면 분식 탐지 성능이 떨어짐
결론
  • 분식회계 탐지에서는 단순한 하이퍼파라미터 튜닝보다 변수 선택(RFECV)과 불균형 처리(SMOTE)가 성능 개선에 결정적
  • NN은 기본 성능이 균형 잡혀 있으나, RF를 전처리와 함께 사용하면 더 높은 성능 가능
  • 최종적으로:
    • 분식 탐지율 극대화 → RFECV + RF
    • 클래스 간 균형 유지 → SMOTE + RF
